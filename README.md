# advent-of-code-js

## Clear, didactic and efficient solutions 

The code is written in the most basic vanilla JavaScript, without external libraries (excepting crypto), trying to be very accessible for everybody.

For some puzzles, **more than one algorithm** is published, allowing one to compare the differences in logic and efficiency.

## Advent of Code 2015 

All days have solutions.

**Day 3** has two alternative solutions (array vs dictionary).

## Advent of Code 2016 

**Day 11** is the first puzzle, since 2015 - Day 1, that makes the computer run out of memory. You cannot use recursion: stack overflow happens. A naive queue becomes too long and makes it slow to search for its content. And part 2 of the puzzle demands really smart optimizations.

Other days are under development.
